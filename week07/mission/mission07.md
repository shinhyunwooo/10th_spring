1. 내가 진행중인 미션 조회하기
구현 방식
이 API는 오프셋 기반 페이지네이션으로 구현했다.
page 값을 기준으로 페이지 단위 데이터를 조회하도록 구성했다.

구현 흐름
- Request Header에서 memberId를 전달받는다.
- Controller에서 memberId, status, page 값을 전달받는다.
- Service에서 해당 사용자의 진행중 미션만 조회한다.
- Repository에서 페이지 조건에 맞는 데이터를 조회한다.
- 조회 결과를 DTO로 변환하여 반환한다.

2. 내가 생성한 리뷰 조회하기
구현 방식
이 API는 일반적인 page 기반이 아닌 커서 기반 페이지네이션으로 구현했다.
이전 조회 데이터의 마지막 값을 커서로 사용하여 다음 데이터를 조회하는 방식으로 처리했다.

구현 흐름
사용자 ID와 커서 값을 요청으로 전달받는다.
- 정렬 기준에 따라 조회 로직을 분기한다.
- ID 기준 조회
- 별점 기준 조회
- 마지막으로 조회된 데이터 이후의 리뷰만 조회한다.
- Slice 기반으로 다음 데이터 존재 여부를 함께 반환한다.
- 응답 DTO로 변환하여 반환한다.

정렬 기준별 구현 내용
1) ID 순 조회
- 리뷰 ID를 기준으로 최신순 조회하도록 구현했다.
- 마지막 리뷰 ID를 nextCursor 값으로 반환하여 다음 조회 시 사용할 수 있도록 처리했다.
2) 별점 순 조회
- 별점을 기준으로 오름차순 / 내림차순 조회가 가능하도록 구현했다.
- 동일 별점 데이터가 존재할 수 있으므로 reviewId를 함께 사용해 안정적으로 조회되도록 처리했다.
- 마지막 리뷰의 별점을 nextStarCursor 값으로 반환하도록 구현했다.

3. Request Body 검증 적용하기
구현 방식

Controller에서는 @Valid를 사용하여 요청 시 자동 검증되도록 구현했다.
