- Page와 Slice
    
    Spring Data JPA에서 대량의 데이터를 나누어 조회(페이징)할 때 사용하는 반환 타입입니다.
    
    - **Page**:
        - **특징**: 전체 데이터 개수를 조회하는 **Count Query**가 추가로 실행됩니다.
        - **장점**: 전체 페이지 수, 전체 데이터 수를 알 수 있어 게시판 하단 숫자 네비게이션(1, 2, 3...)을 만들 때 필수적입니다.
        - **단점**: 데이터량이 많을 경우 Count 쿼리 자체의 성능 부하가 발생할 수 있습니다.
    - **Slice**:
        - **특징**: Count 쿼리를 던지지 않고, 다음 페이지가 있는지 여부만 확인합니다. (조회 시 설정한 `limit + 1`을 조회)
        - **장점**: 성능상 이점이 큽니다. '더보기' 버튼이나 무한 스크롤 UI를 구현할 때 최적입니다.
- Java stream API
    
    Java 8부터 도입된 연속된 데이터(컬렉션, 배열 등)를 처리하기 위한 기능입니다.
    
    - **정의**: 데이터를 추상화하여 다루며, 함수형 프로그래밍 스타일로 선언적인 코드를 작성할 수 있게 해줍니다.
    - **특징**:
        - **파이프라인 구조**: 중간 연산(filter, map, sorted)과 최종 연산(collect, forEach, count)으로 나뉩니다.
        - **지연 연산(Lazy Evaluation)**: 최종 연산이 호출될 때까지 중간 연산은 실행되지 않아 효율적입니다.
        - **불변성**: 원본 데이터를 변경하지 않고 새로운 결과를 생성합니다.
- 객체 그래프 탐색
    
    객체 간의 연관관계를 따라가며 필요한 객체를 찾아가는 과정입니다.
    
    - **개요**: 예를 들어 `Order` 객체가 있을 때, `order.getMember()`를 통해 주문한 회원 객체를 참조하는 것을 말합니다.
    - **핵심 이슈 (JPA)**:
        - **지연 로딩(Lazy Loading)**: 실제 객체를 사용할 때 쿼리를 날려 가져오는 방식.
        - **즉시 로딩(Eager Loading)**: 처음부터 조인을 통해 한꺼번에 가져오는 방식.
        - **주의점**: 연관된 객체가 DB에 로드되지 않은 상태에서 접근하면 `NullPointerException`이나 `LazyInitializationException`이 발생할 수 있습니다.
- @Valid vs @Validated
    
    Spring에서 데이터 검증(Validation)을 위해 사용하는 어노테이션입니다.
    
    | **구분** | **@Valid** | **@Validated** |
    | --- | --- | --- |
    | **표준** | 자바 표준(JSR-303/JSR-380) | Spring 전용 어노테이션 |
    | **사용처** | 주로 Controller의 메서드 인자(Body 등) | Controller, Service, 클래스 레벨 등 |
    | **그룹핑** | **제약 조건 그룹화 불가능** | **그룹화 가능 (Group Validation)** |
    | **동작 방식** | ArgumentResolver에 의해 검증 | AOP를 기반으로 동작 |
    
    단순한 DTO 검증은 `@Valid`를 주로 사용하고, 특정 상황(등록 시 vs 수정 시)에 따라 검증 로직을 다르게 적용해야 할 때는 `@Validated`의 그룹 기능을 사용하면 좋습니다
