- 아키텍처 구조란?
    
    아키텍처 구조는 시스템을 구성하는 요소(컴포넌트)와 이들 간의 관계, 그리고 상호작용 방식을 정의하는 설계 체계를 의미한다.
    
    이는 시스템의 전체적인 틀을 결정하며, 유지보수성, 확장성, 성능, 안정성 등에 직접적인 영향을 미친다.
    
    일반적으로 아키텍처 구조는 다음과 같은 목적을 가진다.
    
    - 시스템 구성 요소의 역할과 책임을 명확히 분리
    - 변경에 유연하게 대응할 수 있는 구조 제공
    - 재사용성과 테스트 용이성 확보
    
    대표적인 아키텍처 구조로는 레이어드 아키텍처(Layered Architecture), 마이크로서비스 아키텍처(MSA) 등이 있다.
    
- Swagger란?
    
    Swagger는 RESTful API를 설계, 문서화, 테스트할 수 있도록 지원하는 오픈소스 도구이다.
    
    현재는 OpenAPI Specification(OAS)이라는 표준을 기반으로 동작하며, API 명세를 JSON 또는 YAML 형식으로 정의한다.
    
    Swagger의 주요 기능은 다음과 같다.
    
    - API 명세 자동 생성
    - API 테스트 UI 제공 (Swagger UI)
    - 서버 및 클라이언트 코드 자동 생성
    
    이를 통해 개발자는 API의 구조를 명확하게 공유할 수 있으며, 프론트엔드와 백엔드 간 협업 효율을 높일 수 있다.
    
- 도메인형 아키텍처란?
    
    도메인형 아키텍처는 비즈니스 로직을 중심으로 시스템을 구성하는 설계 방식이다.
    
    여기서 "도메인"이란 특정 문제 영역(예: 결제, 회원, 주문 등)을 의미한다.
    
    이 아키텍처의 핵심은 다음과 같다.
    
    - 비즈니스 규칙을 중심으로 구조를 설계
    - 기술 중심이 아닌 문제 해결 중심 접근
    - 도메인 단위로 책임을 분리
    
    즉, 시스템을 기능이나 계층이 아닌 비즈니스 의미 단위로 나누는 것이 특징이다.
    
- DDD vs 도메인형 아키텍처
    
    
    | 구분 | DDD | 도메인형 아키텍처 |
    | --- | --- | --- |
    | 정의 | 도메인 중심 설계를 위한 개발 방법론 | 도메인을 기준으로 구조를 나누는 아키텍처 방식 |
    | 범위 | 설계 철학 + 개발 방법론 | 시스템 구조 설계 |
    | 특징 | 유비쿼터스 언어, 엔티티, 밸류 객체 등 포함 | 도메인 기준으로 모듈 분리 |
    | 관계 | 도메인형 아키텍처를 포함하는 상위 개념 | DDD의 일부 구현 방식 |
    
    즉, DDD는 설계 방법론이며, 도메인형 아키텍처는 이를 구현하는 하나의 구조적 접근 방식이라고 볼 수 있다.
    
- 왜 DTO를 사용하는가?
    
    DTO는 계층 간 데이터 전달을 위한 객체이다.
    
    주로 Controller ↔ Service ↔ Repository 간 데이터 교환 시 사용된다.
    
    DTO를 사용하는 주요 이유는 다음과 같다.
    
    1. 데이터 캡슐화
        - 필요한 데이터만 전달하여 불필요한 정보 노출 방지
    2. 계층 간 의존성 감소
        - Entity를 직접 노출하지 않아 구조 변경에 유연하게 대응 가능
    3. 보안 강화
        - 민감한 데이터(예: 비밀번호 등) 외부 노출 방지
    4. API 명세 명확화
        - 요청(Request)과 응답(Response)을 명확히 분리
- 컨버터는 왜 사용하는가?
    
    컨버터는 서로 다른 객체 간의 변환(Entity ↔ DTO 등)을 담당하는 컴포넌트이다.
    
    컨버터를 사용하는 이유는 다음과 같다.
    
    1. 관심사 분리 (Separation of Concerns)
        - 변환 로직을 별도로 분리하여 코드의 역할을 명확히 함
    2. 코드 재사용성 향상
        - 동일한 변환 로직을 여러 곳에서 재사용 가능
    3. 유지보수성 향상
        - 변환 로직 변경 시 해당 클래스만 수정하면 됨
    4. 가독성 향상
        - 서비스 로직에서 불필요한 변환 코드 제거
    
    예를 들어, Entity를 직접 Controller에 반환하지 않고 DTO로 변환하는 과정에서 컨버터가 활용된다.
