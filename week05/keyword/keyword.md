- 빌더패턴이란?
    
    빌더 패턴은 객체 생성 시 생성자의 매개변수가 많아질 경우 가독성과 유지보수성을 개선하기 위해 사용하는 디자인 패턴이다.
    
    필요한 값만 선택적으로 설정할 수 있으며, 객체를 단계적으로 생성할 수 있다는 특징이 있다.
    
    ### 특징
    
    - 생성자 오버로딩 문제 해결
    - 가독성 향상
    - 불변 객체 생성에 유리
    
    ### 장점
    
    - 코드의 가독성이 높아진다.
    - 필수 값과 선택 값을 구분할 수 있다.
    - 객체 생성 과정이 명확해진다.
    
    ### 단점
    
    - 클래스 내부에 빌더 클래스를 추가해야 하므로 코드가 길어진다.
    - 간단한 객체 생성에는 오히려 비효율적일 수 있다.


- record vs static class
    
    ### record
    
    record는 Java에서 불변(Immutable) 데이터 객체를 간단하게 정의하기 위해 사용하는 문법이다.
    
    주로 DTO(Data Transfer Object)에서 많이 사용된다.
    
    ### 특징
    
    - 자동으로 getter, equals, hashCode, toString 생성
    - 불변 객체
    - 생성자 자동 생성
    
    ### 장점
    
    - 코드가 매우 간결해진다.
    - 데이터 전달 객체에 적합하다.
    
    ### 단점
    
    - 필드를 변경할 수 없다.
    - 복잡한 로직에는 부적합하다.
    
    ### static class
    
    static class는 외부 클래스에 종속되지 않고 독립적으로 존재할 수 있는 내부 클래스이다.
    
    ### 특징
    
    - 외부 클래스의 인스턴스 없이 생성 가능
    - DTO나 유틸 클래스에 많이 사용
    
    ### 장점
    
    - 구조적으로 관련된 클래스를 묶을 수 있다.
    - 메모리 효율적이다.
    
    ### 단점
    
    - 남용 시 구조가 복잡해질 수 있다.



- 제네릭이란?
    
    제네릭은 클래스나 메서드에서 사용할 데이터 타입을 컴파일 시점이 아닌 선언 시점에 일반화하여 재사용성을 높이는 문법이다.
    
    ### 예시
    
    ```
    publicclassApiResponse<T> {
    privateTresult;
    }
    ```
    
    ### 특징
    
    - 타입 안정성 보장
    - 캐스팅 제거
    - 재사용성 증가
    
    ### 장점
    
    - 컴파일 단계에서 타입 체크 가능
    - 코드 중복 감소
    
    ### 단점
    
    - 코드가 다소 복잡해질 수 있다.
    - 타입 소거(Type Erasure)로 인해 런타임 타입 정보 제한


- @RestControllerAdvice이란?
    
    @RestControllerAdvice는 Spring에서 전역적으로 예외를 처리하기 위한 어노테이션이다.
    
    Controller에서 발생하는 예외를 한 곳에서 관리할 수 있도록 도와준다.
    
    ### 특징
    
    - 전역 예외 처리
    - JSON 형태로 응답 반환
    - @ExceptionHandler와 함께 사용
    
    ### 장점
    
    - 중복된 예외 처리 코드 제거
    - 일관된 API 응답 구조 유지
    
    ### 단점
    
    - 설정을 잘못하면 모든 예외가 동일하게 처리될 수 있다.
    - 디버깅 시 흐름 파악이 어려울 수 있다.


- Optional이란?
    
    Optional은 값이 존재할 수도 있고, 존재하지 않을 수도 있는 상황을 명확하게 표현하기 위한 클래스이다.
    
    ### 예시
    
    ```
    Optional<String>name=Optional.ofNullable(null);
    ```
    
    ### 특징
    
    - null을 직접 사용하지 않도록 유도
    - 값 존재 여부를 명확히 표현
    
    ### 주요 메서드
    
    - isPresent()
    - orElse()
    - orElseGet()
    - orElseThrow()
    
    ### 장점
    
    - NullPointerException 방지
    - 코드의 안정성 증가
    
    ### 단점
    
    - 남용 시 오히려 코드 가독성 저하
    - 필드나 DTO에는 사용을 권장하지 않음
